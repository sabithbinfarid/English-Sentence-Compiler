# BLOCK Compiler

BLOCK is a block-based language that compiles to C.

Tagline: "No braces. No semicolons. No pointers. Just clean, powerful code."

## Project Layout

- `src/lexer.l` - Flex lexer with token definitions and lexer error handling
- `src/parser.y` - Bison grammar with parsing rules and code generation hooks
- `src/lexer_impl.c` - Hand-written lexer reference implementation
- `src/parser.tab.h` - Token header (normally generated by Bison)
- `examples/*.blk` - 5 sample BLOCK source files
- `Makefile` - Build and utility targets
- `build_flexbison.ps1` - Windows PowerShell build helper for Flex/Bison

## Language Summary

### Variables

```blk
set num score to 95
set decimal gpa to 3.85
set text name to "Rahim"
set x to 42
```

### Conditionals

```blk
when score is greater than 79
    say "Grade A"
otherwise
    say "Grade B"
end when
```

### Loops

```blk
repeat 5 times
    say "Hello"
end repeat

set count to 3
while count is greater than 0
    say count
    set count to count - 1
end while
```

### Functions

```blk
define square(num x)
    give x * x
end define

set y to call square(7)
say y
```

## Flex/Bison Build (Primary)

### Linux/macOS (with make)

```sh
cd block_compiler
make
```

or

```sh
cd block_compiler
make flexbison
```

### Windows PowerShell (no make required)

```powershell
cd block_compiler
./build_flexbison.ps1
```

The script handles common GNUWin32 `bison`/`m4` path issues on Windows.

## Usage

```sh
./blockc examples/01_hello.blk
./blockc examples/05_full.blk --show-ir
./blockc examples/03_loops.blk --run
./blockc examples/05_full.blk --show-ir --run
```

## Features in Flex/Bison Compiler

- Lexical analysis with comment skipping and line tracking
- Syntax parsing for program/when/repeat/while/define/give/say/set/call
- Type system: `num`, `decimal`, `text`
- Type inference for untyped `set` declarations
- Implicit numeric promotion (`int` -> `double`)
- Constant folding for numeric constant expressions
- Smart print format selection in `say`
- `--show-ir` symbol table output
- `--run` compile generated C with `gcc` and execute

## Notes

- `blockc` outputs a `.c` file beside the input `.blk` file.
- Function definitions must appear before main-body statements.
- Error format:
  - Lexer: `[Lexer Error] Line N: ...`
  - Parser/Syntax: `[Syntax Error] Line N: ...`
